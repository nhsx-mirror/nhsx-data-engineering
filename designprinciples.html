Azure Design Principles¶

Data Engineering Concepts¶

Design Patterns¶

  1. Abstract and fully perameratised

  2. Reusable code

  3. Tested and robust

  4. Maintainable

  5. Efficient use of resources

Ingestion¶

Standardised and parameterised ingestion templates for every scenario. Allow us to rapidly ingest new datasets with minimal configuration

Utilites¶

Utilities can be reusable code or logic app APIs that allow us to standardise specific functions called across multiple pipelines. Most involve sending data back and forth to systems outside data factory and/or updating configuration files

Config as Code¶

  1. All pipelines are controlled from a JSON configuration file

  2. Centralises the orchestration

  3. Avoids config mismatches between activities

  4. Adds git version control

Docs as Code¶

  1. Principle that you should be writing documentation with the same tools as your code

  2. Take it one step further - documentation is auto generated from the code

  3. No way for documentation to be outdated or misaligned
